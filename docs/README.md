# OctoAcme Project Management Docs

Welcome to the OctoAcme project management documentation hub. This README is the central index for all process documents, helping teams manage projects efficiently and consistently.

## Overview

OctoAcme follows a structured, iterative project management approach grounded in five core principles:

- **Customer-first** — prioritize customer value and usability in every decision.
- **Iterative delivery** — deliver small, testable increments rather than big-bang releases.
- **Clear ownership** — each project has a named Project Manager (PM) and Product Lead accountable for outcomes.
- **Data-informed decisions** — measure impact and iterate based on evidence, not assumptions.
- **Psychological safety** — encourage open feedback, experimentation, and learning from failure.

## Lifecycle Stages

| Stage | Purpose |
|-------|---------|
| **Initiation** | Align stakeholders, define the problem statement, and create the project charter. |
| **Planning** | Scope the work, map the backlog, identify risks, and set milestones. |
| **Execution & Tracking** | Build, test, review, and iterate; track progress against the plan. |
| **Release & Deployment** | Pre-release verification, deployment, rollback handling, and announcement. |
| **Retrospective & Continuous Improvement** | Capture learnings and drive process improvements. |

Risk management and communication practices run across all stages of the lifecycle.

## Core Roles

- **Project Manager (PM)** — coordinates delivery, schedules, risk, and stakeholder communications.
- **Product Manager (PdM)** — defines outcomes, prioritizes the backlog, and measures success.
- **Developers** — implement features and collaborate on design, testability, and code reviews.
- **QA / Testing** — validate quality and acceptance criteria before release.
- **Stakeholders** — provide inputs, approvals, and strategic direction.

4. **Approaching release?** Use the [Release & Deployment](octoacme-release-and-deployment.md) checklist.
